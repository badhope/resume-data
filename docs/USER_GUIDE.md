# 简历数据清洗系统 v2.0

## 一、系统概述

简历数据清洗系统是一款智能化的简历解析与数据处理工具，能够自动从各种格式的简历文件中提取结构化信息，并进行数据清洗和标准化处理。系统支持 PDF、DOCX、TXT、HTML 等多种格式，提供直观的 Web 界面，让简历管理变得简单高效。

### 核心特性

- **智能解析**：基于 NLP 技术自动提取简历关键信息
- **多格式支持**：支持 PDF、DOCX、TXT、HTML 等常见格式
- **数据清洗**：自动标准化日期、学历、公司名称等字段
- **敏感信息处理**：支持脱敏、保留、删除三种处理模式
- **批量操作**：支持批量上传、批量导出、批量删除
- **统计分析**：实时展示解析质量和系统使用情况
- **自定义配置**：灵活配置清洗选项和导出模板

## 三、功能模块详解

### 3.1 上传中心

#### 功能说明
上传中心是简历处理的入口，支持单文件和批量上传。

#### 操作步骤
1. 点击左侧菜单「上传简历」进入上传页面
2. 拖拽文件到上传区域，或点击选择文件
3. 系统自动识别文件格式并开始解析
4. 解析完成后显示结果，可查看详情或继续上传

#### 支持格式
| 格式 | 扩展名 | 说明 |
|------|--------|------|
| PDF | .pdf | 推荐格式，解析效果最佳 |
| Word | .docx | 支持 Office 2007+ 格式 |
| 文本 | .txt | 纯文本格式 |
| 网页 | .html | HTML 格式简历 |

#### 注意事项
- 单个文件大小不超过 10MB
- 批量上传建议不超过 20 个文件
- 确保简历内容为中文或英文

---

### 3.2 简历管理

#### 功能说明
简历管理模块提供简历列表查看、详情浏览、编辑修改、导出下载等功能。

#### 简历列表
- **分页浏览**：支持 10/20/50/100 条每页
- **搜索筛选**：按文件名、姓名筛选
- **批量选择**：勾选多条记录进行批量操作
- **快速操作**：查看、导出、删除

#### 简历详情
点击「查看」进入详情页，可查看：

**基本信息**
- 姓名、性别、出生日期
- 联系电话、电子邮箱
- 现居住地

**教育经历**
- 学校名称
- 专业方向
- 学历层次
- 就读时间

**工作经历**
- 公司名称
- 职位名称
- 工作时间
- 工作描述

**其他信息**
- 专业技能
- 证书资质
- 项目经验
- 自我评价

#### 在线编辑
- 点击「编辑」按钮进入编辑模式
- 修改字段后点击「保存」
- 支持撤销和重置操作

#### 导出功能
支持三种导出格式：
- **JSON**：结构化数据，适合程序处理
- **Excel**：表格格式，适合人工查看
- **CSV**：通用格式，兼容性强

---

### 3.3 数据统计

#### 功能说明
数据统计模块展示系统使用情况和解析质量分析。

#### 统计指标
| 指标 | 说明 |
|------|------|
| 简历总数 | 系统中存储的简历总量 |
| 解析成功 | 成功解析的简历数量 |
| 平均可信度 | 解析结果的可信度平均值 |
| 今日上传 | 当日上传的简历数量 |
| 本周上传 | 近 7 天上传的简历数量 |
| 成功率 | 解析成功数占总数的比例 |

#### 字段分析
展示各字段的识别成功率：
- 姓名、性别、手机、邮箱等基本信息
- 教育经历、工作经历等复杂字段
- 技能、证书等其他信息

#### 趋势图表
展示近期的上传趋势，帮助了解系统使用情况。

---

### 3.4 系统设置

#### 功能说明
系统设置模块提供清洗选项配置、导出模板管理、操作历史查看功能。

#### 清洗选项

**字段提取设置**
可单独开关各字段的提取功能：
- 基本信息：姓名、性别、出生日期、手机、邮箱、地址
- 经历信息：教育经历、工作经历、项目经验
- 其他信息：专业技能、证书资质、自我评价

**数据处理设置**
- **敏感信息处理**：
  - 保留原始数据：不做处理
  - 脱敏处理（推荐）：部分隐藏敏感信息
  - 完全移除：删除敏感字段
- **日期格式**：
  - 标准格式：2020-09
  - 中文格式：2020年9月
  - 斜杠格式：09/2020
- **学历名称标准化**：统一学历表述（如"本科"统一为"大学本科"）
- **公司名称标准化**：去除公司名称中的冗余信息

#### 导出模板
管理系统预设和自定义的导出模板：
- 查看模板列表
- 创建自定义模板
- 删除非默认模板

#### 操作历史
记录用户的操作日志：
- 配置更新
- 简历上传/解析/删除
- 数据导出

## 四、API 接口文档

### 4.1 基础信息
- **Base URL**: `http://localhost:8000/api`
- **Content-Type**: `application/json`
- **响应格式**: JSON

### 4.2 接口列表

#### 文件上传
```
POST /upload
Content-Type: multipart/form-data

参数：
- file: 文件对象

响应：
{
  "file_id": "string",
  "filename": "string",
  "file_path": "string"
}
```

#### 批量上传
```
POST /upload/batch
Content-Type: multipart/form-data

参数：
- files: 文件对象数组

响应：
{
  "results": [
    {
      "file_id": "string",
      "filename": "string",
      "file_path": "string"
    }
  ]
}
```

#### 解析简历
```
POST /resume/parse/{file_id}

响应：
{
  "resume_id": "string",
  "data": { ... },
  "confidence": 0.95,
  "parse_time": "2024-01-01T12:00:00"
}
```

#### 获取简历列表
```
GET /resume?skip=0&limit=10

响应：
{
  "total": 100,
  "items": [ ... ]
}
```

#### 获取简历详情
```
GET /resume/{resume_id}

响应：
{
  "resume_id": "string",
  "filename": "string",
  "data": { ... },
  "confidence": 0.95,
  "parse_time": "2024-01-01T12:00:00"
}
```

#### 更新简历
```
PUT /resume/{resume_id}

参数：
{
  "data": { ... }
}

响应：
{
  "message": "更新成功"
}
```

#### 删除简历
```
DELETE /resume/{resume_id}

响应：
{
  "message": "删除成功"
}
```

#### 导出简历
```
GET /export/json/{resume_id}
GET /export/excel/{resume_id}
GET /export/csv/{resume_id}

响应：文件流
```

#### 批量导出
```
GET /export/batch/json?resume_ids=id1,id2,id3
GET /export/batch/excel?resume_ids=id1,id2,id3

响应：文件流
```

#### 获取清洗选项
```
GET /config/options

响应：
{
  "options": { ... },
  "updated_at": "2024-01-01T12:00:00"
}
```

#### 更新清洗选项
```
PUT /config/options

参数：
{
  "extract_name": true,
  "sensitive_mode": "mask",
  ...
}

响应：
{
  "options": { ... },
  "updated_at": "2024-01-01T12:00:00"
}
```

#### 获取统计数据
```
GET /config/statistics

响应：
{
  "overview": { ... },
  "field_stats": [ ... ],
  "trend_data": [ ... ]
}
```

## 五、部署指南

### 5.1 环境要求
- Python 3.9+
- Node.js 16+
- npm 或 yarn

### 5.2 后端部署

```bash
# 进入后端目录
cd backend

# 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Linux/Mac
# 或 venv\Scripts\activate  # Windows

# 安装依赖
pip install -r requirements.txt

# 启动服务
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### 5.3 前端部署

```bash
# 进入前端目录
cd frontend

# 安装依赖
npm install

# 开发模式
npm run dev

# 生产构建
npm run build
```

### 5.4 Docker 部署

```bash
# 使用 Docker Compose
docker-compose up -d

# 访问地址
# 前端：http://localhost:3005
# 后端：http://localhost:8000
```

## 六、常见问题

### Q1: 上传文件后解析失败？
**A**: 请检查：
- 文件格式是否支持
- 文件是否损坏
- 文件内容是否为中文或英文

### Q2: 解析可信度较低？
**A**: 可能原因：
- 简历格式不规范
- 关键信息缺失
- 文件编码问题

建议使用标准格式的 PDF 文件。

### Q3: 如何批量导出？
**A**: 
1. 在简历列表页面勾选需要导出的简历
2. 点击「批量导出」按钮
3. 选择导出格式

### Q4: 敏感信息如何处理？
**A**: 在「系统设置」-「清洗选项」中配置：
- 脱敏处理：手机号显示为 138****1234
- 完全移除：删除所有敏感字段

### Q5: 如何自定义导出模板？
**A**: 
1. 进入「系统设置」-「导出模板」
2. 点击「新建模板」
3. 配置模板名称、字段、格式
4. 保存后即可使用
